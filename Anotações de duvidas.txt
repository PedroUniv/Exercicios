O que tem em um site?

HTML é focada em conteudo, focada em imagem, texto, listas, videos e etc.
Css é focado em design , tipo cores, sombras, tamanhos, posicionamentos e etc.
JavaScript é focado em interações, menus interarivos, popups, validações e etc.

//////////Cores////////// 

(O uso minimo para criar uma paleta é 3 e no maximo 5 cores, NUNCA ultrapassar esses valores, se não o site pode ficar muito poluido)
Cores primarias : Amarelo, Vermelho e Azul
Cores secundarias : Laranja , Violeta e Verde
Cores terciarias : Amarelo-esverdeado, Amarelo-alaranjado, Vermelho-alaranjado, Vermelho-arroxeado, Azul-arroxeado, Azul-esverdeado.

//////////Temperaturas de cores//////////

Cores Quentes: Amarelo, Amarelo-alaranjado, Laranja, Vermelho-alaranjado, Vermelho, Vermelho-arroxeado

Cores Frias: Roxo, Azul-arroxeado, Azul, Azul-esverdeado, Verde, Amarelo-esverdeado

Cores Complementares: Sempre usar a cor oposta no "circulo cromatico" ----- Bom usar quando se tem um fundo de uma cor e o texto de outra cor

Cores Análogas: Sempre usar as cores ao lado no "circulo cromatico"

Cor Análogas e complementares: Junção das duas tecnicas de cores que da uma harmonia e uma cor de constraste.

Cores Análogas Relacionadas: Pegar duas cores vizinhas, pula uma cor ao lado e perga a terceira cor do circulo cromatico , por exemplo Amarelo, Amarelo-alaranjado, (L̶a̶r̶a̶n̶j̶a̶), Vermelho-alaranjado. No caso usariamos apenas o Amarelo, Amarelo-alaranjado e Vermelho-alaranjado.Posso escolher o lado que vou usar para esquerda ou direita.

Cores Intercaladas: Posso usar tambem para intercalar as cores, por exemplo Amarelo, (A̶m̶a̶r̶e̶l̶o̶-̶a̶l̶a̶r̶a̶n̶j̶a̶d̶o̶), Laranja, (V̶e̶r̶m̶e̶l̶h̶o̶-̶a̶l̶a̶r̶a̶n̶j̶a̶d̶o̶), Vermelho. Fica uma paleta mais seca e dura mas em alguns casos pode ser bom utilizar.

Cores Triadicas: É uma tecnica que se usa pulando três cores o circulo cromatico, usando num formato 3x3, selecionando 3 cores.

Cores em quadrado: É uma tecnica que se usa pulando duas cores no circulo cromatico 2x2, selecionando 4 cores.Usado para escolher cores fortes e bem balanceadas.

Cores tetradicas: Isso é uma tecnica que faz a junção de 2 cores complementares , por exemplo Verde>Vermelho e Laranja>Azul , assim formando uma paleta de 4 cores, que conversam entre si.

Monocromia: São cores modificadas pela saturação e luminosidade de uma cor. Pode ser usado para fazer degradê.

(Um bom site para poder ver as paletas de cores mais perfeitamente é o Adobe Colors / Paletton / Coolors)

Como eram feito os livros antigamente?
R= Os monge copistas , pegavam os livros originais, e reescreviam em livros em brancos e eles demoravam meses para serem feitos, porque eles eram copiados á mão.

O Gutenberg e sua maquina no Seculo XV.
R=Ele criou uma Prensa de Tipos Móveis, que era uma maquina usada a tinta para ajudar os Copistas, mas a maquina você colocava o papel em baixo , selecionava letra por letra e por cima ia uma prensa com tinta e assim saía a copia perfeita do livro.

A tipografia vem da palavra grega TIPO =  týpos / GRAFIA = graphía , que significa o estudos de tipos ou das letras.

//////////Anatomia dos Tipos//////////

Tipo = como vou desenhar cada uma das letras, como vou demonstrar o significado do produto.
Sempre que vão criar uma fonte nova ele começam pela letra "x" por causa do tamanho das outras letras, eles chamam isso de (altura-x), e para as letras maiusculas tem o nome de (Altura das maiusculas), e as letras minusculas que vazam um pouco e são quase do tamanho das maiusculas tem o nome de (Ascedente), e tem o nome dos vazamentos para baixo que se chamam (Descendente), e a soma de todas as alturas se chamam (Corpo).

//////////Anatomia do Tipo//////////

Serifa = todas as pontinhas das letras , são um pequeno tracinho que tem em algumas fontes, e elas servem para criar uma linha imaginaria que nosso cerebro cria que facilita a leitura das palavras

//////////Anatomia Geometrica//////////

Haste = seria meio que a base de uma letra escrita.
Filete = é a parte que une duas hastes ou um arco.
Esporão = Algumas fontes tem isso para meio que elas ter uma base de onde ficarem apoiada.
Terminal = um elemento que não esta preso em outro lugar, pode ser facilmente confundido com braços e penas das letras.
Braço = Sai da haste e se eleva pra cima.
Perna = As vezes sai da haste e as vezes sai do braço e isso se chama perna
Pé = Vem seguinte á perna ou pode se chamar tambem de apoio da letra
Algumas letras tem barriga  por exemplo a letra "b", outras ja tem uma Espinha por exemplo a letra "s".
Olho = a parte vazia dentro das letras 
Orelha = a parte de cima de uma letra um mini traço que existe na letra "g" (Não sao todas a fontes que tem isso).
Cauda = aquelas sobrinhas que existem na parte de baixo de uma letra por exemplo na letra "g e q".
Glifo = 1 letra
Fonte = Conjunto de uma determinada familia de Glifos, elas tem varias formas que são as seguintes, light, normal, seminegrito, negrito e extranegrito, geralmente todas as fontes tem normal e negrito, mas tem algumas que nem tem negrito é so o normal mesmo, mas algumas outras tem essa variação  de familias tipograficas.

//////////Categoria de fonte//////////

(Caso esteja com duvidas sobre combinações de fonte, é so acessar "CSS web safe font combinations, site W3Schools.com)
(Nunca esquecer de usar site para textos com preto e branco para fundos ou textos.)
Fonte Serifada = Tem serifa, recomendado para textos impressos.
Fonte Sans-Serif = Não tem serifa, recomendado para sites com leitura muito prolongada.
Fonte Monoespaçada = Todos os Glifos tem a mesma largura, mas pode ser com Serifa ou sem Serifa.
Fonte HandWriting = Tenta simular a escritura à mão
Fontes Comemorativas ou Fontes Display = Não se preocupa a obedecer as regras anatomicas, são fontes comemorativas geralmente são fontes que te levam ideia a filmes por ex StarWars, HarryPotter, Fortnite, GodOfWar e etc...

Posso retirar fontes de um site por exemplo o GoogleFonts, pego a fonte em formato EMBED e em @import, e adiciono em meu arquivo CSS (sempre no começo do codigo) por exemplo :
"@import url('https://fonts.googleapis.com/css2?family=Roboto:ital,wght@0,100;0,300;0,400;0,500;0,700;0,900;1,100;1,300;1,400;1,500;1,700;1,900&display=swap');"
E sempre lembrar de adicionar a "font-family" que ja vem junto na pagina onde pego o codigo da fonte por exemplo: font-family: "Roboto", sans-serif; , esse é a tag de font-family dessa fonte em especifica.

//////////Modelos de caixa//////////

São aplicados em HTML e em CSS, é um conjunto que facilita a organização por partes do site, eu posso acabar colocando uma caixa dentro da outra e da outra e etc...Esse termo é chamado de aninhamento. Todo modelo visivel em um site é em modelo de caixa

height: Altura
width: Largura
Quando se junta esses dois o termo usado é Box-sizing

display:inline; - Usando isso eu transformo um modelo de box-level em um modelo inline-level. Se eu adicionar esse codigo eu não posso ter height e width
display:block; - Usando isso eu transformo um modelo inline-level em um modelo box-level.
display:inline-block; - Tem as caracteristicas de um inline e block , assim posso mudar o tamanho e largura mesmo ele sendo inline.

border: Borda que fica dentro da caixa

margin-left: 10px;

margin: auto; - Isso é usado caso eu queira centralizar uma "caixa" no meio da tela

outline: Contorno ou Traçado que fica foca do elemento (Muito parecido com a border, e sempre vai ficar entre a "border e margin")

border-width: espessura da borda

//////////Tipos de caixa//////////

box-level:
Sempre se inicia em uma linha nova
Sempre ocupa a largura total da tela
Essa tags são do tipo box-level: <div>, <h1>-<h6>, <p>, <main>, <header>, <nav>, <article>, <aside>, <footer>, <form>, <video>.
O <div> é mais usado em box-level


inline-level:
Não pula para a proxima linha,sempre pula pro lado do texto terminado, do tamanho exato pra conter o conteudo do elemento,e ele continua o conteudo sem quebra de linha, diferente do box-level.
Essas tags são do tipo inline-level: <span>, <a>, <code>, <small>, <strong>, <em>, <sub>-<sup>, <label>, <button>, <input>, <select>
O <span> é mais usado em inline-level

user agent = configuração feita pelo navegador

display: block; = Elemento do estilo box-level. a parte do (block) é box-level

////////// Formularios //////////

Todo formulario tem que estar entre as tag <form>

Todo formulario tem um mini script de javascript rodando nele.

Para criar uma caixa de texto basta usar a tag <input type="text" name="" id="">
Para criar um botão basta usar a  tag <input type="submit" value=""> - No (value="Texto") tenho que adicionar uma palavra ao botão.

Não existe uma ligação entre o texto e a caixa de texto por enquanto.

Posso desabilitar as sugestões de login usando <form autocomplete="off"> dentro da primeira tag do form.

Posso mandar um cadastro usando  <form action="NomeDoArquivo.(Linguagem)">  

name="" é mais util para HTML e PHP .
id="" é mais util para JavaScript e uso do label.

- SEMPRE relacionar o <label> com o meu input -
Para eu poder fazer uma ligação/relação  neles eu tenho que envelopar minha palavra do formulario:

 <p><label for="inome">Nome:</label>
 <input type="text" name="nome" id="inome"></p>


Existem 2 metodos simples , para envio de formulario

1 Metodo = ou metodo padrão e o metodo GET - method="GET" - que coloca os dados na URL
<form action="cadastro.php" (method="GET")>

2 Metodo = POST - que ja é o metodo que não fica os dados na URL
<form action="cadastro.php" method="POST">

Mas mesmo assim os dados continuam visiveis, mas não na url, e tem uma maneira de guardar esses dados
Na pagina do formulario basta clicar em botão direito > inspecionar (ou apertar CTRL + SHIFT + I) e depois clicar em Network > Headers.

Mas quando usar GET ou POST
GET = quando os dados não forem sensiveis por exemplo, NOME, SOBRENOME, PESO, ALTURA e etc, e o GET pode enviar até 3mil bytes (3mil letras).
POST = quando os dados forem sensiveis por exemplo  Numero do cartão de credito , CEP, Endereço, Fotos e etc, e o POST pode ultrapassar os 3mil bytes.
